# ilist v0.1.6

## Highlights

- Multiple independently authorized Google My Drive mounts with custom names, paths, and optional root folder IDs.
- Streamed ordinary downloads with single-range support.
- Google Docs PDF/DOCX, Sheets PDF/XLSX, and Slides PDF/PPTX exports, including PDF-first preview.
- Small and resumable uploads plus folder creation, rename, move, and trash operations.
- Workspace export actions in desktop menus, mobile action sheets, and controlled shares.

## Deployment

1. Back up the production D1 database outside Git.
2. Enable Google Drive API and create a Web OAuth client.
3. Register `https://YOUR_ILIST_ORIGIN/api/admin/oauth/google/callback` as an authorized redirect URI.
4. Configure `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` as Worker secrets.
5. Keep `PUBLIC_ORIGIN` set to the exact deployed HTTPS origin without a trailing slash.
6. Apply all D1 migrations, run `npm run check`, and deploy.

See [Google Drive Setup](../google-drive-setup.md) for the complete OAuth and mount procedure.

## Limits

Google support is currently limited to My Drive. Shared Drives, Shared with me, and shortcut traversal are not included. OAuth applications serving users outside a testing allowlist may require Google verification.
